📦 Installation

Install the package via NPM and sync your project.

npm install capacitor-call-manager
npx cap sync

⚠️ Critical: Manual Setup Required

Android requires these permissions to be explicitly declared in your Main App's manifest. Capacitor will not auto-sync these.

<!-- android/app/src/main/AndroidManifest.xml -->
<uses-permission android:name="android.permission.READ_CALL_LOG" />
<uses-permission android:name="android.permission.READ_PHONE_STATE" />
<uses-permission android:name="android.permission.SYSTEM_ALERT_WINDOW" />
<uses-permission android:name="android.permission.POST_NOTIFICATIONS" />

🛠️ Quick Start

Initialize the plugin and request necessary permissions in your main application logic.

import { CallManager } from 'capacitor-call-manager';

async function setup() {
  // Request required Android permissions
  const status = await CallManager.requestPermissions();
  
  if (status.overlay !== 'granted') {
    // Required to show the CRM popup
    await CallManager.requestOverlayPermission();
  }

  // Activate background monitoring
  await CallManager.startCallListener();
}

🎭 Multi-Entity Tracking

Categorize your numbers to show different UIs for Leads, Customers, or Employees.

await CallManager.addTrackedNumbers({
  items: [
    { 
      number: '+1234567', 
      name: 'John Doe', 
      entityType: 'SALES', 
      entityId: 'lead_99' 
    },
    { 
      number: '+9876543', 
      name: 'Sarah Smith', 
      entityType: 'CRM', 
      entityId: 'cust_45' 
    }
  ]
});

When Sarah calls, your overlay will receive ?entityType=CRM&entityId=cust_45 in the URL query string.

📚 API Reference

Method Return Type Description
addTrackedNumbers Promise<{success, count}> Syncs numbers with metadata (Entity/ID) to the native DB.
getTrackedNumbersByEntityId Promise<{success, items}> Fetch tracked items by specific unique entity ID.
getCallLogs Promise<{success, logs}> Retrieves system call history with advanced filters.
removeTrackedNumbersByEntityId Promise<{success, count}> Surgical cleanup of local database by Entity ID.
getPendingSubmissions Promise<{success, submissions}> Fetches offline user interactions from offline mode.
setBackgroundServiceEnabled Promise<{success}> Toggles Global vs App-Only mode for background task.
